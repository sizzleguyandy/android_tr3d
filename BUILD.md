# Virgin Race — APK Build Guide

## Prerequisites

- **Node.js 18+** — https://nodejs.org
- **Android Studio** (latest stable) — https://developer.android.com/studio
- **Java 17** (bundled with Android Studio, or install via SDKMAN)
- **Android SDK** with API 33+ (install via Android Studio SDK Manager)

---

## Step 1 — Configure the API server URL

Edit `www/app.js`, find the `CFG` object near the top and update:

```js
const CFG = {
  API: "https://YOUR_SERVER_ADDRESS:8000",   // ← replace this
  ...
};
```

---

## Step 2 — Install dependencies

```bash
cd virgin-race-app
npm install
```

---

## Step 3 — Add the Android platform

```bash
npx cap add android
```

This creates the `android/` folder with the native Android project.

---

## Step 4 — Sync web assets to Android

```bash
npx cap sync android
```

Run this every time you change files in `www/`.

---

## Step 5 — Open in Android Studio

```bash
npx cap open android
```

Android Studio will open the project. On first open:
- Let Gradle sync finish (may take a few minutes)
- If prompted, update Gradle plugin version — accept

---

## Step 6 — Build the APK

### Debug APK (for testing / side-loading)

In Android Studio:
1. **Build → Build Bundle(s) / APK(s) → Build APK(s)**
2. APK will be at: `android/app/build/outputs/apk/debug/app-debug.apk`

Or via command line:
```bash
cd android
./gradlew assembleDebug
```

### Release APK (for distribution)

```bash
cd android
./gradlew assembleRelease
```

You'll need a signing keystore. Generate one:
```bash
keytool -genkey -v -keystore virgin-race.keystore \
  -alias virgin-race -keyalg RSA -keysize 2048 -validity 10000
```

Add signing config to `android/app/build.gradle`:
```gradle
signingConfigs {
    release {
        storeFile file('../virgin-race.keystore')
        storePassword 'YOUR_STORE_PASSWORD'
        keyAlias 'virgin-race'
        keyPassword 'YOUR_KEY_PASSWORD'
    }
}
buildTypes {
    release {
        signingConfig signingConfigs.release
        minifyEnabled false
    }
}
```

## White-labelling for a different client

To rebrand for a different fitness company, edit `www/app.js`:

```js
const CFG = {
  API:       "https://your-server:8000",
  brandName: "Client Race",        // Tab label and page titles
  brandColor: "#FF6600",           // Primary accent colour (hex)
  logoUrl:   "assets/client-logo.png",  // Logo shown on home/onboarding
  tabLabel:  "Client Race",        // Bottom tab button text
};
```

Update `android/app/src/main/res/values/strings.xml`:
```xml
<string name="app_name">Client Race</string>
```

Update the `appId` in `capacitor.config.json`:
```json
{
  "appId": "com.clientname.race",
  "appName": "Client Race"
}
```

## Troubleshooting

**API calls fail on device (CORS / cleartext)**
- Ensure `cleartext: true` is in `capacitor.config.json` (already set)
- For HTTPS, your server needs a valid SSL cert
- For local testing over HTTP, add `android:usesCleartextTraffic="true"` in `android/app/src/main/AndroidManifest.xml` — Capacitor should handle this automatically with `allowMixedContent: true`

**White screen after launch**
- Run `npx cap sync` to ensure latest web assets are copied
- Check Android Studio Logcat for JS errors (filter by `chromium`)
- Enable WebContents debugging in `capacitor.config.json` (`"webContentsDebuggingEnabled": true`) then use `chrome://inspect` to debug in Chrome DevTools

**Gradle sync fails**
- File → Invalidate Caches and Restart in Android Studio
- Ensure Android SDK API 33+ is installed (SDK Manager → SDK Platforms)
